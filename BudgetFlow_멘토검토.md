# BudgetFlow 서비스 기획 검토 요약

**Team BudgetFlow** | 클라우드컴퓨팅 기말프로젝트 | 2026.05

---

## 서비스 한 줄 정의

> 슬랙 채널에 영수증과 지출 설명을 보내면 AI가 자동으로 분석·분류하고, 관리자가 지정한 엑셀 양식에 맞춰 예산안 또는 지출내역서를 생성하는 **팀 단위 예산 정산 자동화 서비스**

---

## 해결하는 문제

동아리·학회·행사 운영진이 예산 정산 시 겪는 3가지 고통:

1. 영수증을 메신저에서 수집해 엑셀에 수기 입력하는 반복 작업
2. 여러 명이 각자 결제 → 회계 담당자가 흩어진 증빙을 수작업으로 취합
3. 기관마다 다른 엑셀 양식에 맞춰 매번 재작성

---

## 시스템 구성 및 역할

```
[팀원]
  슬랙 채널에 영수증 이미지 / 텍스트 지출 입력
          ↓
  ┌───────────────────┐
  │   슬랙 봇 담당     │  Webhook 수신, 즉시 접수 확인 메시지 응답
  └────────┬──────────┘
           ↓ (비동기 전달)
  ┌───────────────────┐
  │   백엔드 담당      │  분석 파이프라인 실행, DB 저장, 결과 전달
  └────────┬──────────┘
           ↓
  ┌───────────────────┐
  │  LLM/OCR 담당     │  텍스트 파싱, 영수증 OCR, 카테고리 분류, 신뢰도 계산
  └────────┬──────────┘
           ↓
  ┌───────────────────┐
  │ 프론트엔드 담당   │  관리자 대시보드, 검토·수정, 엑셀 다운로드
  └───────────────────┘
```

## 핵심 설계 결정

| 항목              | 결정                                                 | 근거                                                   |
| ----------------- | ---------------------------------------------------- | ------------------------------------------------------ |
| 조직 구조         | Organization → Project 계층                          | 동아리는 매 학기 행사 반복, 양식·카테고리 재사용 필요  |
| 메신저            | 슬랙 우선, 카카오톡 MVP 제외                         | 카카오 그룹 봇 정책 제약, 안정적인 Slack Bolt 우선     |
| 사용자 온보딩     | Slack Workspace 연동, 팀원 별도 가입 없음            | 가입 요구 시 실사용률 급감, UX 마찰 최소화             |
| 슬랙 응답 방식    | 즉시 접수 확인 + 비동기 분석 결과 전송               | Webhook 3초 제한, OCR+LLM 처리는 10~30초 소요          |
| 엑셀 양식 파싱    | LLM 자동 컬럼 매핑 추천 + 관리자 검토 필수           | 자동 확정 시 금액 열 오매핑으로 잘못된 엑셀 생성 위험  |
| 예산 매핑 방식    | 카테고리 단위 소진 추적                              | 품목 1:1 매핑은 LLM 오류 누적 시 데이터 오염           |
| needs_review 조건 | AI 신뢰도 낮음 / 예산 초과 / **영수증 없음(무조건)** | 텍스트 단독 제출은 지출증빙 불가, 관리자 확인 필수     |
| 프로젝트 마감     | 관리자 수동 [정산 마감] 클릭                         | 동아리 특성상 늦은 제출 多, 자동 마감 시 운영 혼란     |
| 엑셀 생성 정책    | needs_review 있어도 생성 가능, 경고 후 제외          | 마감 당일 미처리 항목으로 전체 제출이 막히는 상황 방지 |
| 예산안·지출내역서 | 독립 생성 가능                                       | 예산안 없이 먼저 집행 후 정산하는 케이스 다수          |

## MVP 단계별 범위

**MVP 1차 — 핵심 흐름 동작 검증**

| 역할       | 구현 내용                                                                    |
| ---------- | ---------------------------------------------------------------------------- |
| 봇         | 슬랙 Webhook 수신, 즉시 접수 응답, 분석 결과 메시지, 관리자 멘션 알림        |
| 백엔드     | 조직/프로젝트/지출 데이터 관리, 비동기 분석 파이프라인, Slack Workspace 연동 |
| LLM/OCR    | 텍스트 지출 파싱, 카테고리 자동 분류, 신뢰도 계산, needs_review 판정         |
| 프론트엔드 | 로그인, 프로젝트 관리, 지출 목록 조회, 검토·수정, 엑셀 다운로드              |

**MVP 2차** — 영수증 OCR, 엑셀 양식 업로드 및 LLM 컬럼 매핑, 카테고리 예산 한도 설정

**MVP 3차** — 상품 링크 분석, 카카오톡 연동, 통계 대시보드

## 검토 요청 사항

1. **엑셀 양식 LLM 파싱 신뢰성** — 병합 셀·다중 헤더 구조의 실제 학교 양식에서 Bedrock이 컬럼 구조를 정확히 추론할 수 있는지, 실패 시 fallback 전략이 충분한지
2. **비동기 파이프라인 비용** — Lambda + Step Functions + Textract + Bedrock 조합의 건당 처리 비용이 학생 프로젝트 예산 범위에서 허용 가능한 수준인지
3. **Slack Workspace 연동 범위** — 팀원 별도 가입 없이 Slack 사용자 정보를 가져올 때 필요한 OAuth 스코프와 데이터 보관 정책
