BF BudgetFlow 내부 시연용 프로토타입

Admin workflow prototype

Slack 지출 입력부터 관리자 검토, 엑셀 다운로드까지 이어지는 운영 도구 화면입니다.

각 화면은 독립 HTML 파일로 분리되어 있습니다. 데스크톱에서는 높은 정보 밀도를, 모바일에서는 하단 탭과 카드형 검토 흐름을 기준으로 설계했습니다.

/login

로그인

데스크톱은 서비스 맥락을, 모바일은 입력과 로그인 버튼만 남긴 진입 화면입니다.

 /projects

프로젝트

행사 단위 예산, Slack 채널, 엑셀 양식 상태와 생성 폼을 배치했습니다.

 /expenses

지출 목록

목록과 우측 검토 패널, 승인/반려/엑셀 생성 흐름을 가장 완성도 높게 구성했습니다.

 /settings

설정

엑셀 양식, LLM 컬럼 매핑, 예산 카테고리와 분류 키워드를 관리합니다.
Slack 접수팀원은 채널에 설명과 증빙을 제출
비동기 분석LLM이 금액, 카테고리, 위험 신호를 추출
검토 필요신뢰도 낮음, 증빙 없음, 예산 초과 가능성 표시
관리자 승인필드 수정 후 승인 또는 반려 처리
정산 마감수동으로 프로젝트 정산을 닫음
엑셀 생성승인 완료 항목만 포함해 다운로드

시각 시스템은 흰색/neutral 표면, near-black CTA, 얇은 border, 8-12px radius, 상태색 중심으로 고정했습니다. 장식용 그래픽보다 금액, 상태, 검토 사유, 다음 액션이 가까이 보이도록 설계했습니다.
